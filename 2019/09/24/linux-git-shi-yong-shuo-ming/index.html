Git 配置

  1. 配置工具

    • 使用git config进行配置
    • 使用--system选项配置/etc/gitconfig文件，适用于所有用户
    • 使用--global选项配置~/.gitconfig文件，适用于当前用户
    • 不使用任何选项，则配置当前项目目录中的.git/config文件，适用于当前项目

  2. 配置个人用户信息

     git config --global user.name "用户名"     #设置用户名
 git config --global user.email "邮箱"      #设置邮箱

  3. 配置默认文本编辑器

     git config --global core.editor vim

  4. 配置差异分析工具

     git config --global merge.tool vimdiff

    可选择的有kdiff3, tkdiff, meld, xxdiff, emerge, vimdiff, gvimdiff, ecmerge, opendiff等

  5. 查看配置信息

    git config --list

    重复的变量名说明来自不同的配置文件，以最后一个为准

创建Git仓库

  1. 创建本地仓库
     git init #以当前目录作为Git仓库
 git init newrepo #以newrepo目录作为Git仓库
  2. 克隆远程仓库到本地
     git clone <repo> <local directory> # 克隆repo到本地目录，省略本地目录则克隆到当前目录

    远程repo的三种写法(SSH协议速度最快)

     git clone git@github.com:schacon/grit.git    #SSH协议
 git clone git://github.com/schacon/grit.git   #GIT协议
 git clone https://github.com/schacon/grit.git   #HTTPS协议

Git基本操作

  1. 添加文件到缓存
     git add <文件1> <文件2> <文件3>
 git add .   #添加所有新增和修改到缓存，忽略删除文件操作
 git add -u   #添加所有删除和修改到缓存，忽略新增文件操作
 git add -A   #添加所有新增、修改和删除到缓存，忽略删除文件操作
  2. 提交缓存区内容到仓库
     git commit -m '提交信息'
 git commit -am '提交信息'   #跳过添加缓存，直接提交文件
  3. 查看状态
     git status
 git status -s   #以简短结果输出
  4. 查看缓存改动
     git diff   #查看尚未缓存的改动
 git diff --cached   #查看已缓存的改动
 git diff HEAD   #查看尚未缓存与已缓存的所有改动
 git diff --stat   #仅显示摘要
  5. 取消已缓存内容
     git reset HEAD <file name>
  6. 移除文件
     git rm <file name>   #移除未缓存本地文件
 git rm -f <file name>   #移除已缓存的本地文件
 git rm --cached <file name>   #仅从缓存区移除文件（保留本地文件）
 git rm -r <directory name>   #递归删除目录
  7. 移动/重命名文件
     git mv <file name> <new neme/directory>

Git分支管理

```
git branch <branch name>   #创建本地分支

git branch   #查看本地分支

git checkout <branch name>   #切换本地分支

git checkout -b <branch name>   #创建并立即切换到新分支

git checkout -d <branch name>   #删除分支

git merge <branch name>   #将选定分支合并到当前分支
```

查看Git提交历史

```
git log

git log --oneline   #查看简洁版历史记录

git log --graph   #以拓扑图形式查看历史记录

git log --reverse   #逆向查看历史记录

git log --author   #查看指定用户的提交记录

git log --since(or --before, --until, --after)={3.weeks.ago    (or 2019-04-31)}   #指定日期查看

git log --no-merges   #隐藏合并提交
```

多个命令可以一起使用，例如：
git log --reverse --oneline

Git标签

```
git tag -a <tag name>   #创建标签

git tag -a <tag name> -m <tag information>   #创建带注解的标签

git tag -a <tag name> <commit number>   #追加标签

git tag   #列出所有标签

git tag -s <tag name> -m <tag information>   #PGP签名标签

git tag -d <tag name>   #删除标签

git show <tag name>   #查看某标签对应版本的修改内容
```

Git远程仓库（以Github为例）

  1. 链接github 

     ssh-keygen -t rsa -C "github注册邮箱"   #生成SSH Key

    进入~/.ssh文件夹，打开id_rsa.pub文件，复制里面的SSH public key到github网站

     ssh -T git@github.com   #验证是否链接成功

  2. 查看当前远程库

     git remote
 git remote -v   #显示每个别名的实际链接地址

  3. 提取远程仓库

     git fetch <远程库> <branch name> #从指定远程库中拉取指定分支，拉取下来的分支名为<远程库/分支名>

 git merge <远程库/分支名>   #将拉取下来的分支合并到当前分支

 git checkout -d <远程库/分支名>   #删除拉取下来的分支

    上述步骤可以简化成下面一步：

     git pull <远程库> <branch name>   #将远程库中的指定分支拉取并与本地当前分支合并

    git pull <远程库> <远程分支名>:<本地分支名> #将远程库中的指定分支拉取并与指定本地分支合并
    ```

  4. 推送本地分支到远程仓库

     git push <远程库> <branch name>   #将指定的本地分支推送到指定远程库中的同名分支

 git push <远程库> <本地分支名>:<远程分支名>   #将指定的本地分支推送到指定远程库中的指定分支

  5. 添加远程库 

     git remote add <远程库> <url>

  6. 删除远程仓库 

     git remote rm <远程库>
